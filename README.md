# Прогнозирование IC50, CC50 и SI по RDKit-дескрипторам

Проект выполнен в рамках задачи машинного обучения для анализа данных, предоставленных химиками. Цель работы - построить и сравнить модели, позволяющие прогнозировать эффективность и безопасность соединений по молекулярным RDKit-дескрипторам, а также определить наиболее качественные решения для регрессионных и классификационных постановок.

## Постановка задачи

На основании предоставленных данных необходимо построить прогноз, позволяющий подобрать наиболее эффективное сочетание параметров для создания лекарственных препаратов.

В рамках проекта решаются следующие задачи:

1. Регрессия для `IC50, mM`.
2. Регрессия для `CC50, mM`.
3. Регрессия для `SI`.
4. Классификация: превышает ли значение `IC50, mM` медианное значение выборки.
5. Классификация: превышает ли значение `CC50, mM` медианное значение выборки.
6. Классификация: превышает ли значение `SI` медианное значение выборки.
7. Классификация: превышает ли значение `SI` порог `8`.

Для каждой задачи были построены и сравнены несколько моделей машинного обучения. Итоговый выбор моделей обоснован на основе метрик качества, анализа ошибок, кросс-валидации, тюнинга гиперпараметров и интерпретации признаков.

## Краткое описание данных

В работе используются молекулярные RDKit-дескрипторы и три целевые переменные:

- `IC50, mM` - полумаксимальная ингибирующая концентрация;
- `CC50, mM` - цитотоксическая концентрация 50%;
- `SI` - индекс селективности, рассчитываемый как `SI = CC50 / IC50`.

После очистки данных итоговая выборка содержит 966 наблюдений. В EDA выявлены правосторонняя асимметрия целевых переменных, экстремальный хвост у `SI` и выраженная мультиколлинеарность части дескрипторов.

## Последовательность запуска ноутбуков

Важно запускать ноутбуки в правильном порядке, потому что моделирующие ноутбуки используют `.pkl`-файлы, которые создаются на этапах EDA и preprocessing.

### 1. EDA

Сначала запускается:

```text
SF_ML_EDA.ipynb
```

На этом этапе выполняются:

- первичная загрузка данных;
- проверка размерности датасета;
- анализ пропусков и дубликатов;
- анализ распределений `IC50`, `CC50`, `SI`;
- анализ баланса классов для классификационных задач;
- анализ корреляций и мультиколлинеарности;
- сохранение промежуточных данных для дальнейшей обработки.

### 2. Preprocessing

Затем запускается:

```text
SF_ML_preprocessing.ipynb
```

Этот ноутбук создаёт основные `.pkl`-файлы, необходимые для всех последующих моделей:

```text
SF_ML_data_tree.pkl
SF_ML_data_linear.pkl
```

Назначение файлов:

- `SF_ML_data_tree.pkl` - набор признаков после удаления константных и высококоррелированных признаков, предназначенный для tree-based моделей;
- `SF_ML_data_linear.pkl` - набор признаков после дополнительной VIF-редукции, предназначенный для линейных моделей и моделей, чувствительных к мультиколлинеарности.

Если запустить ноутбуки регрессии или классификации до preprocessing, они не найдут необходимые `.pkl`-файлы.

### 3. Регрессионные модели

После preprocessing можно запускать регрессионные ноутбуки:

```text
SF_ML_regression_IC50.ipynb
SF_ML_regression_CC50.ipynb
SF_ML_regression_SI.ipynb
```

В этих ноутбуках выполняются:

- загрузка `SF_ML_data_tree.pkl` и `SF_ML_data_linear.pkl`;
- исключение целевых переменных из признакового пространства;
- логарифмирование целевой переменной через `log1p`;
- обучение baseline-моделей;
- кросс-валидация;
- тюнинг гиперпараметров;
- оценка на тестовой выборке;
- анализ ошибок и важности признаков;
- сохранение лучших моделей.

### 4. Классификационные модели

После preprocessing также можно запускать классификационные ноутбуки:

```text
SF_ML_classification_IC50_gt_median.ipynb
SF_ML_classification_CC50_gt_median.ipynb
SF_ML_classification_SI_gt_median.ipynb
SF_ML_classification_SI_gt_8.ipynb
```

В этих ноутбуках выполняются:

- формирование бинарной целевой переменной;
- стратифицированное train/test-разбиение;
- обучение baseline-моделей;
- кросс-валидация;
- анализ `overfit_gap`;
- тюнинг гиперпараметров;
- оценка по `ROC_AUC`, `PR_AUC`, `F1`, `recall`, `precision`, `accuracy`, `balanced_accuracy`;
- построение confusion matrix, ROC-кривой и PR-кривой;
- сохранение лучших моделей.

Для задач по медиане основной метрикой является `ROC_AUC`. Для задачи `SI > 8` основной метрикой является `PR_AUC`, так как положительный класс составляет около 35.4% выборки.

## Предотвращение утечки данных

При моделировании из признакового пространства исключаются все целевые переменные:

```text
IC50, mM
CC50, mM
SI
```

Это особенно важно для задач, связанных с `SI`, потому что `SI` рассчитывается как `CC50 / IC50`. Использование `IC50`, `CC50` или `SI` как признаков привело бы к утечке данных и завышению качества моделей.

## Использованные подходы

В работе сравнивались следующие группы моделей:

- наивные baseline-модели: `DummyRegressor`, `DummyClassifier`;
- линейные модели: `LinearRegression`, `Ridge`, `Lasso`, `ElasticNet`, `LogisticRegression`;
- устойчивые линейные модели: `HuberRegressor`;
- SVM-модели: `SVR`, `SVC`, `LinearSVC`;
- ансамбли деревьев: `RandomForest`, `ExtraTrees`;
- бустинговые модели: `GradientBoosting`, `HistGradientBoosting`, `XGBoost`, `LightGBM`, `AdaBoost`.

Для подбора гиперпараметров использовался `RandomizedSearchCV` с кросс-валидацией.

## Основные результаты

### Регрессия

| Задача | Финальная модель | Основной результат | Комментарий |
|---|---|---:|---|
| IC50 | XGBoost baseline | `R2_original = 0.377` | умеренное качество, ошибки выше в правом хвосте |
| CC50 | ExtraTrees baseline | `R2_original = 0.529` | лучшая регрессионная задача |
| SI | GradientBoosting_tuned | `RMSE_log = 1.226` | сложная производная цель, исходная шкала нестабильна |

### Классификация

| Задача | Финальная модель | Основная метрика | F1 | PR_AUC |
|---|---|---:|---:|---:|
| IC50 > median | RandomForest_tuned | `ROC_AUC = 0.818` | 0.735 | 0.818 |
| CC50 > median | ExtraTrees_tuned | `ROC_AUC = 0.831` | 0.745 | 0.834 |
| SI > median | ExtraTrees_tuned | `ROC_AUC = 0.754` | 0.659 | 0.752 |
| SI > 8 | ExtraTrees_tuned | `PR_AUC = 0.652` | 0.586 | 0.652 |

Лучшие результаты получены для задач, связанных с `CC50`. Задачи для `SI` оказались наиболее сложными из-за производного характера целевой переменной и экстремального правого хвоста распределения.

## Выводы

1. RDKit-дескрипторы содержат устойчивый предиктивный сигнал для всех поставленных задач.
2. Наиболее успешно моделируется `CC50`: и в регрессии, и в классификации получены лучшие результаты.
3. `IC50` предсказывается умеренно, но классификация `IC50 > median` показывает хорошее качество.
4. `SI` является наиболее сложной целью, поскольку зависит от двух экспериментальных величин и имеет экстремальный правый хвост.
5. Tree-based модели чаще показывают лучшие результаты, но склонны к переобучению на небольшой выборке.
6. Для линейных моделей важна VIF-редукция, так как исходные RDKit-дескрипторы имеют выраженную мультиколлинеарность.
7. Для задачи `SI > 8` важно использовать `PR_AUC`, а не только `accuracy`, так как положительный класс менее частый.

## Ограничения исследования

Основные ограничения:

- небольшой размер выборки после очистки: 966 наблюдений;
- выраженные правые хвосты целевых переменных;
- сильная мультиколлинеарность исходных RDKit-дескрипторов;
- риск переобучения tree-based моделей;
- отсутствие внешней валидационной выборки;
- feature importance является модельной оценкой, а не доказательством химической причинности.

## Рекомендации по дальнейшему улучшению

1. Расширить обучающую выборку до 2000-3000 соединений.
2. Добавить молекулярные fingerprints, например ECFP4 / Morgan fingerprints.
3. Для `IC50` и `CC50` использовать `R2_original` как вторичную метрику при подборе гиперпараметров.
4. Для `SI` дополнительно контролировать качество на log-шкале и ошибки в хвосте распределения.
5. Для `SI > 8` выполнить подбор порога классификации для снижения числа false negative.
6. Рассмотреть альтернативную постановку: предсказывать `IC50` и `CC50` отдельно, а затем вычислять `SI`.
